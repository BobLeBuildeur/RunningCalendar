# Design principles

These are the key UX/UI principles that must **always** be observed when designing and implementing the RunningCalendar interface.

## 1. Clarity over density

Race discovery should feel effortless—even with complex filters and large datasets.

## 2. Speed is a feature

Interactions (search, filters, map) must feel instant.

## 3. Action-oriented UI

Every screen should guide users toward: discover → evaluate → register → track.

## 4. Progressive disclosure

Show essential race info first; reveal advanced details on demand.

## 5. Data-first design

UI components are built to handle structured, dynamic, and sometimes incomplete data gracefully.
